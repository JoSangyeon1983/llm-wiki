# 위키 문서 작성 지침 (WRITING_GUIDE)

> **TL;DR** — 위키 문서는 ① 쉽게 ② 읽는 이 처지에서 ③ 정확하게 씀.
> 작성은 **준비 → 작성 → 검토** 3단계로, 마지막에 §6 점검표로 자가 검토함.
> 이 파일은 LLM(Claude)이 위키 문서를 쓰거나 고칠 때 **반드시 준수**하는 작성 규약임.
> CLAUDE.md `§4 문서 작성 규칙`의 상세판임. 문체·구조 규칙(두괄식·개조식·음슴체)은 CLAUDE.md를 따르고,
> 이 파일은 "좋은 문장·표현"의 기준을 정의함.

---

## 1. 세 가지 요건 (한눈에)

| 요건 | 요소 | 자가 점검 질문 |
|------|------|----------------|
| **쉽게 쓰기** | 형식의 용이성 | 쉬운 말·정연한 구성으로 자연스럽게 읽히는가? 시각적 편의를 고려했는가? |
| | 내용의 용이성 | 분량이 적절한가? 내용을 논리적으로 연결했는가? |
| **읽는 이 처지에서 쓰기** | 고압적 표현 삼가 | 명령·권위적 표현을 쓰지 않았는가? |
| | 차별적 표현 삼가 | 성·지역·연령·인종·장애 등 차별·비하 표현이 없는가? |
| **정확하게 쓰기** | 표기의 정확성 | 오·탈자가 없는가? 어문 규범에 맞는가? |
| | 표현의 정확성 | 의미에 맞는 단어인가? 어법에 맞는 문장인가? |

> 위키의 "독자"는 **미래의 나**와 **위키를 다시 읽을 사용자**임. 늘 그 처지에서 씀.

---

## 2. 쉽게 쓰기

### 가. 자연스러운 표현 사용하기
1. **쉬운 말로 쓰기** — 어려운 용어 대신 이해하기 쉬운 표현. 어문 규범에 맞춰 한글로 씀.
   외국 글자·한자는 쓰지 않되, 꼭 필요하면 괄호 병기.
2. **줄임말(약어) 남용 금지** — 글은 짧아 보여도 처음 보는 사람에겐 어려움.
   꼭 써야 하면 **첫 등장 시 본디 용어를 밝힘**. 예) `연구·개발(R&D)`, `사업 모델(BM)`.
3. **외래어·외국어·한자어 남용 금지** — 쉽고 친숙한 우리말로.
   예) `다운받다→내려받다`, `뉴노멀→새 일상`, `콘텐츠→영상/화상`, `제고함→높임`, `소정의→정해진`, `타 기관→다른 기관`, `동 사업→이 사업`.
4. **우리말다운 표현 쓰기** — 번역 투 피함. 과도한 **피동·사동** 금지.
   예) `변경됨에 의해→변경됨`, `환기를 시켜야→환기를 해야`.
5. **중복·중의 표현 삼가** — 같은 뜻 반복하지 않고, 한 가지 뜻으로만 읽히게 씀.
   예) `사전 동의가 반드시 필요→사전 동의가 필요`.
6. **불필요한 표현 덧붙이지 않기** — 군더더기는 의미를 흐리고 의도를 왜곡함.

### 나. 시각적 편의 고려하기
1. **한눈에 알 수 있게 구성** — 주제와 뒷받침 내용이 **계층적**으로 드러나게. 긴 서술은 항목화(불릿).
2. **가독성 있게 배열** — 핵심을 앞에(두괄식). 강조는 **굵게**·밑줄·`작은따옴표`. 표·그림 등 보조 자료 활용.

## 3. 내용의 용이성

### 가. 적절한 분량으로 제시하기
- 문장은 **짧고 간결·명료**하게. 길어지면 여러 문장으로 나눔.
- 한 문단에 다 담지 말고 단락을 적절히 나눔.
- 필요한 정보를 생략하지 않음 — 접속·경로 등은 **구체적으로**.

### 나. 논리적으로 연결하기
- **대등한 것끼리 접속** — 접속어 앞뒤 의미를 맞춤.
- **조사·어미를 정확하게** — 부적절한 조사나 과도한 생략 금지.
- 주제 관련 내용을 유기적으로 묶어 **논리적으로 단락 구성**.

---

## 4. 읽는 이 처지에서 쓰기

### 가. 고압적 표현 삼가기
- 명령·권위적 표현은 거부감을 줌. 완곡하게 바꿈.
  - `~할 것` → `~하세요` / `~해야 함`
  - `~바람` → `~하기 바람`(위키는 음슴체이므로 지시문 자체를 최소화)
  - 명령형 `~하라/~어라` → `~함` / `~하면 됨`
  - `~다 할 것이다` → `~다` (장황한 표현은 서술어로 간결하게)

### 나. 차별적 표현 삼가기
- 성·지역·연령·인종·국가·장애 등에 대한 **차별·비하 표현 금지**.
- 누구나 알기 쉬운 용어 사용.
- **문맥상 필요 없는 인적 정보**(특정 개인·집단 지목)는 넣지 않음.

---

## 5. 정확하게 쓰기

### 가. 표기의 정확성
- **오·탈자 없게** 꼼꼼히 점검.
- **한글 맞춤법·표준어 규정** 준수. 예) `년도→연도`, `아니오→아니요`(가부 응답).
- **띄어쓰기 규정** 준수. 예) `1천만원→1천만 원`, `기재여부→기재 여부`.
- **외래어 표기법** 준수. 예) `마켓팅→마케팅`.

### 나. 표현의 정확성
- **의미에 맞는 단어** 사용. 예) `일체↔일절`, `~확인용→~확인하는 용도`.
- **어법에 맞는 문장** — 문장 성분 간 **호응**(주어-서술어, 목적어-서술어).
  예) `~로의 전환 또는 희망하는` → `~로 전환하려는 또는 이를 희망하는`.

## 6. 작성 3단계 + 검토 점검표

### 단계 1 — 준비하기
- **독자 파악**: 누구인가 / 주제에 대해 무엇을 아는가 / 무엇을 알아야 하는가 / 어떤 의문을 가지는가.
  (위키에선 "미래의 나"가 나중에 무엇을 찾을지 상정함.)
- **목적·유형 결정**: 개념(정의·원리)인지, 실용(HowTo)인지, 요약(reference)인지 → PARA 폴더·`type` 결정.
- **자료 수집·분석**: `sources/`·기존 페이지에서 근거 확보 → 신뢰성·시의성 확인 → 핵심 정보 추림.

### 단계 2 — 작성하기
- §2~§5 요건을 적용해 본문 작성. 두괄식·개조식·음슴체(CLAUDE.md §4).
- 새 개념은 `[[ ]]`로 연결, 주장엔 `(출처: source-id)` 표기.

### 단계 3 — 검토하기 (자가 점검표)

**① 쉽게 썼는가?**
- [ ] 다듬은 말(순화어)을 썼는가 / 번역 투·외래어·한자어 남용은 없는가
- [ ] 중복·중의·불필요한 성분은 없는가
- [ ] 한눈에 알게 구성하고 가독성 있게 배열했는가
- [ ] 문장·문단 분량이 적절한가 / 접속·조사로 논리적으로 연결했는가

**② 읽는 이 처지에서 표현했는가?**
- [ ] 명령·권위적(고압적) 표현을 완곡하게 했는가
- [ ] 차별·비하 표현, 불필요한 인적 정보는 없는가

**③ 정확하게 썼는가?**
- [ ] 오·탈자가 없는가 / 맞춤법·띄어쓰기·외래어·로마자 표기법에 맞는가
- [ ] 의미에 맞는 단어인가 / 문장 성분이 호응하는가

---

## 7. 헷갈릴 때 도움받을 곳
- **국립국어원 온라인가나다** — 어문 규범·어법·표준국어대사전 문의 (☎ 1599-9979).
- **표준국어대사전 / 우리말샘** — 표준어·표기 확인.
- **국립국어원 한국어 어문 규범** — 맞춤법·띄어쓰기·외래어/로마자 표기법 원문.
