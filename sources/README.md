# Sources — 원본 레지스트리

> ★ 불변(immutable) 레이어. LLM은 **읽기만** 함. 원본 파일은 수정하지 않음.

이 폴더는 위키의 모든 주장이 역추적되는 **원본 저장소**임.
새 원본을 추가할 때마다 아래 인덱스 표에 **source-id**를 등록함.

## 등록 규칙

1. 원본 파일(논문 PDF, 기사 캡처, 데이터, 메모 등)을 이 폴더에 둠.
2. 아래 표에 한 행을 추가함. **source-id는 한 번 부여하면 변경하지 않음.**
3. source-id 네이밍: `src-NNN` (3자리 일련번호) 또는 의미 있는 슬러그(예: `karpathy-llm-wiki`).
4. 위키 문서에서 인용 시 본문에 `(출처: source-id)`, 프론트매터에 `sources: [source-id]`로 표기함.

## 등록 전 체크리스트 (사람이 직접 점검)

> 위키의 환각 방지는 **원본 품질이 바닥**임 — 검수(§7)는 "문서↔원본 일치"만 보지 "원본이 참인가"는 못 봄.
> 그건 **사람만** 거를 수 있음. 넣기 전에 아래를 확인함.

- [ ] **신뢰도** — 저자·매체 확인. 출처 불명·미검증 자료는 보류하거나 한계를 비고에 명시함.
- [ ] **시의성** — 발행일 확인. 오래된 자료는 "원본 발행일"을 비고에 적어 stale 판단 근거를 남김.
- [ ] **중복** — 같은 내용이 이미 등록됐는지 인덱스에서 검색함. **갱신본이면 기존 걸 고치지 말고 새 id로 추가**(불변).
- [ ] **포맷** — LLM이 읽히는 형식인가. Word(.docx)·스캔 이미지 PDF는 **텍스트/발췌 동봉** 권장.
- [ ] **라이선스/민감정보** — 인용·공개 가능한지, 개인정보·기밀 포함 여부 확인.
- [ ] **source-id 부여** — 규칙에 맞게 1개 부여(한 번 정하면 불변).
- [ ] **불변 동결** — 등록 후 파일을 수정·삭제하지 않음. 내용이 바뀌면 새 id(예: `src-3-v2`)로 추가함.

## 엔트리 템플릿

인덱스 표에 추가할 한 행(복사해서 채움):

```
| src-NNN | <제목> | <유형> | <원본 경로 또는 URL> | YYYY-MM-DD(등록일) | <원본 발행일·한계·비고> |
```

## source-id 인덱스

| source-id | 제목 | 유형 | 원본 위치/URL | 등록일 | 비고 |
|-----------|------|------|---------------|--------|------|
| _(없음)_ | — | — | — | — | 첫 소스 등록 대기 |

> 유형 예시: paper, article, video, dataset, note, thread, docs
